# Agent Roles

## PM

Owns scope, task breakdown, and task board updates. PM should read `AGENTS.md`, `workspace/PROJECT_CONTEXT.md`, and `workspace/TASK_BOARD.md` first.

## Designer

Owns mobile visual quality, Streamlit polish, Home card presentation, AI Assistant readability, and My Page mock UX. Designer avoids RAG files.

## Developer

Owns implementation of product behavior inside allowed files. Developer preserves RAG behavior and avoids protected RAG files unless explicitly requested.

## QA

Owns test execution, behavior verification, regression review, and final readiness notes. QA should not edit files unless asked.

## Docs

Owns README and docs updates. Docs should not modify application behavior.

## Server / Git

Manual operations pane for running Streamlit, checking status, running smoke checks, and preparing manual commits.
